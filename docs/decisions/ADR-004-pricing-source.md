# ADR-004 · 가격은 자체 event_seat_prices 단일 진실원 + 장르 티어

- 상태: Accepted
- 날짜: 2026-07-02
- 슬라이스: S04(좌석·가격)
- 관련: [[seat]], [[event]]

## 맥락
예매엔 **좌석 등급별 정형 가격**이 필요한데, KOPIS는 이를 주지 않는다. KOPIS가 주는 건
`pcseguidance`(=priceText) **자유텍스트 하나뿐**이고, 값이 들쭉날쭉하다:
`전석 무료` / `가격 미정` / `예매처 확인` / `R석 110,000원 / S석 88,000원` / `10,000원 ~ 50,000원`.

## 결정
1. **결제 가격의 유일 진실원 = `event_seat_prices`**(이벤트×등급 절대 가격). 목록·상세·좌석선택·결제
   모두 이 테이블 기준 → 화면 간 가격 불일치 없음.
2. **KOPIS priceText로 결제 가격을 만들지 않는다.** 상세의 "원 공연 안내" 블록에 원문만 표시
   (무료/미정/숫자없음은 블록 숨김). 결제엔 미사용.
3. 가격 생성 = **장르 기반 티어 + eventId 기반 fallback**. 등급 배수 고정(A=1.0/S=1.3/R=1.6/VIP=2.0).
   시딩 시 절대값 계산·저장, 같은 eventId는 항상 같은 값(멱등, `unique(event_id,grade)`).
4. `base_price = 등급 최저가`를 시딩 단계에서 events에 write → 목록 "가격 미정" 해소.

## 고려한 대안
- **priceText 첫 숫자 파싱 + 배수(초기 A안)**: `R석 110,000 / S석 88,000`에서 첫 숫자(110,000, R석)를
  base로 오인해 VIP가 220,000으로 폭등, `10,000~50,000` 범위는 최저/할인/일부석 구분 불가,
  `무료/미정/확인`은 숫자 없음 → **신뢰 불가로 철회**.
- **전 공연 고정 템플릿**: 파싱 0이나 모든 공연 가격이 동일해 좌석 등급 UI·데모 의미 약화.
- **장르 티어(채택)**: 파싱 리스크 0 + 공연별 가격 차이 + 등급 구조 유지 + stable(멱등).

## 결과 / 한계
- 우리 가격이 KOPIS 원문과 다를 수 있음 → 상세에서 "예매 기준 가격(우리)"과 "원 공연 안내(KOPIS)"를
  라벨로 분리해 혼란 완화. 무료 공연도 데모상 유료로 처리(문서 명시).
- 장르 매핑은 KOPIS `genrenm` 정확 문자열 기준, 미매핑은 `eventId % N`.

## 한 줄
> KOPIS는 공연 메타·자유텍스트 가격만 주고 좌석 등급별 정형 가격/재고는 주지 않는다.
> 따라서 결제 가격은 priceText를 파싱하지 않고 FlowTicket의 event_seat_prices를 단일 진실원으로
> 두며, 시딩 시 장르 티어로 등급 가격을 멱등하게 생성한다. priceText는 참고 안내로만 노출한다.
