# TS-009 · 단위/통합은 초록인데 버그는 수동으로만 잡혔다 — E2E(Playwright) 도입

- 슬라이스: 횡단(S01~S04)
- 날짜: 2026-07-07
- 유형: 검증 전략/회고 (테스트 하네스)
- 관련: [TS-001](TS-001-queue-reentry-after-expiry.md)·[TS-005](TS-005-s04-exception-flow-verification.md)·[TS-006](TS-006-seat-hold-error-not-surfaced.md)·[TS-007](TS-007-release-hold-status-lost-context-clear.md)·[TS-008](TS-008-hold-cross-event-seat-ownership.md), 규칙: [e2e-rules.md](../testing/e2e-rules.md)

> 특정 버그 하나가 아니라, "왜 자꾸 사람이 손으로만 버그를 찾았나"의 회고와 그 해결(E2E 도입).

## 1. 증상 (반복된 패턴)
정적 가드(하네스 계약 24 메타)·단위·통합(Testcontainers) 테스트는 **전부 초록**이었다. 그런데
S03~S04를 진행하는 동안 실제로 화면을 눌러보면 계속 결함이 나왔고, **매번 사람이 수동으로**
발견했다:
- 입장 만료 후 재예매가 막힘 (TS-001)
- 선점 실패(MAX)가 화면에 무음 처리 (TS-006)
- 선점 취소를 눌러도 화면이 안 바뀜 (TS-007의 FE 측)
- 교차 이벤트 좌석 선점 가능 (TS-008)
- 매진/대기만료 화면이 실제 트리거에서 뜨는지 미검증 (TS-005)

즉 **"검증하는 사람 = 개발자(수동)"** 상태였고, 이게 개발 속도의 병목이었다.

## 2. 조사 (왜 자동 테스트가 못 잡았나)
발견된 버그들의 **공통점**: 함수/컴포넌트 하나의 문제가 아니라 **경계 결함**이었다 —
FE↔BE 계약, 프록시/SSR, SSE 반영, 라우팅, 클라이언트 상태(useState) 전이.

- 단위 테스트: 함수/컴포넌트를 jsdom에서 검증 → 실제 브라우저·API 왕복이 없음.
- 통합 테스트(Testcontainers): 백엔드 동시성/상태기계는 잘 눌렀지만(IMP-003/004/007) **브라우저를
  거치지 않음** → "선점 취소 버튼이 실제로 좌석 화면으로 돌아가는가" 같은 건 볼 수 없다.
- 테스팅 트로피에서 **최상위 E2E(브라우저)가 비어 있었다.** 그 계층의 결함은 사람이 클릭해야만 드러났다.

## 3. 근본 원인
**독립적 검증 수단(E2E)의 부재.** 경계·UI 상태 결함을 자동으로 재현할 방법이 없어서,
(1) 발견이 사람의 수동 클릭에 의존했고 (2) 한 번 고쳐도 **회귀 방지 장치가 없었다**(다음에 또
깨질 수 있음). AI 에이전트로 변경 속도가 빨라질수록 이 병목은 더 커진다.

## 4. 해결 — Playwright E2E 하네스 도입
크리티컬 유저 플로우만 대상으로 E2E를 세웠다(전 페이지 X).
- **P0**: 스캐폴딩 + 홈 스모크(로봇이 실제로 도는지).
- **P1**: 예매 해피패스(상세→대기열→좌석→선점). "순간이동"(실제 API로 로그인·대기열 상태를 미리
  세팅)으로 테스트 독립·속도 확보.
- **P2**: **과거 수동으로 잡던 버그를 회귀 테스트로 박제** — 선점 취소 복귀(TS-007), 1인 초과
  안내(TS-006), 제출 직전 매진 리다이렉트(SOLD_OUT 배선).

원칙(= e2e-rules.md): 하드 대기 금지(조건 대기), 시맨틱/콘텐츠 셀렉터, 직렬 실행으로 공유 좌석
경합 제거, 실 PG/SMS 미연동(시뮬레이터/mock).

부수 효과: P2 작성 중 병렬 실행에서 여러 워커가 같은 좌석을 다투자 정확히 SOLD_OUT이 떴다 —
**초과판매 방어(IMP-003)가 실제로 작동함을 E2E가 재확인**하기도 했다.

## 5. 재발 방지
- 손으로 잡던 결함이 이제 **CI에서 자동 검증**된다(P3 게이팅 완료 — ci.yml `e2e` 잡이 PR마다
  풀스택을 띄워 5개 크리티컬 플로우를 실행, 실패 시 머지 차단 + trace 아티팩트 업로드). 다시 깨지면 막힌다.
- 자가 개선 루프: 실패 시 Playwright **trace(CCTV)** 를 에이전트가 파싱해 원인 분석·수정.
- 규율: 새 크리티컬 플로우/새 예외 흐름을 만들면 E2E를 함께 추가한다.
- 교훈: **"CI 초록 ≠ 사용자 경험 정상."** 경계·UI 상태는 실제 브라우저로만 확신할 수 있다.
