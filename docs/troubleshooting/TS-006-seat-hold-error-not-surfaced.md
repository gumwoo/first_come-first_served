# TS-006 · 좌석 선점 실패가 화면에 안 뜸 — MAX_PER_USER_EXCEEDED 등 무음 처리

- 슬라이스: `S04`(좌석·재고)
- 날짜: 2026-07-06
- 유형: UX 결함(프론트)
- 관련 커밋/PR: PR #67
- 관련 문서: [TS-005](TS-005-s04-exception-flow-verification.md)

> 순서: 증상 → 조사 → 근본 원인 → 해결 → 재발 방지.

## 1. 증상
좌석 2매를 선점한 뒤 다시 3매를 선택해 "선택 완료"를 누르면, 서버는
`409 MAX_PER_USER_EXCEEDED("1인 구매 가능 수량을 초과했습니다.")`를 정확히 반환하는데
**화면에는 아무 반응이 없다.** 사용자는 왜 안 되는지 알 수 없음.

## 2. 조사
좌석 페이지 `complete()`의 catch 분기:
```
if (SOLD_OUT) → /sold-out 리다이렉트
else if (QUEUE_NOT_ADMITTED) → /queue 리다이렉트
else { await refresh(); setSelected([]); }   // ← 그 외 전부 무음
```
`MAX_PER_USER_EXCEEDED`(및 `HOLD_EXPIRED`, 이미 선점된 좌석 등)는 마지막 `else`로 떨어져
**재고 갱신 + 선택 초기화만 하고 사용자에게 메시지를 안 띄운다.** 그래서 "눌렀는데 아무 일도
안 일어남"으로 보임.

- 클라이언트 토글은 "이번 선택 ≤ 4매"만 막는다. 이미 보유한 HELD 수는 모르므로
  (기존 2매 + 이번 3매 = 5 > 4) 서버에서만 걸리고, 그 결과가 화면에 전달되지 않았다.

## 3. 근본 원인
**실패 코드 중 리다이렉트 2종만 처리하고 나머지는 사용자 피드백 없이 삼켰다.** 에러 상태와
표시 UI가 없었다.

## 4. 해결
- 에러 상태(`errorMsg`) 추가, 선점 실패 시 사용자에게 안내 배너 노출.
- `MAX_PER_USER_EXCEEDED`: 선택은 유지하고 메시지만(줄여서 재시도 가능).
- 그 외(`HOLD_EXPIRED`/이미 선점 등): 재고 갱신 + 선택 초기화 + 서버 메시지 노출.
- 좌석을 다시 토글하면 메시지 해제.

## 5. 재발 방지
- **실패 응답의 코드마다 UX 매핑을 명시**(리다이렉트/인라인 안내/무시)하고, 기본 분기는
  "무음"이 아니라 "사용자에게 안내"가 되도록 한다.
- API가 반환하는 `error.message`(한글)를 그대로 안내에 활용해 문구 중복 관리를 줄임.

## 한계 / 남은 것
- 토스트가 아니라 사이드바 인라인 배너로 노출. 다중 에러 큐잉/자동 소멸(타임아웃)은 범위 밖.
- 클라이언트가 유저의 기존 보유 수를 선제 표시하는 것은 후속(현재는 서버 판정 + 안내로 충분).
