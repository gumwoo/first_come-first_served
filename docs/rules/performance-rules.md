# Performance Rules

정적 하네스로 검증 불가 → **부하 테스트(k6) 결과로 관리**. (S08 슬라이스)

## 1. 기준 지표 (목표 임계)
| 지표 | 기준 |
|---|---|
| 예매/대기열 핵심 API p95 | ≤ 300ms (부하 시) |
| 에러율 | ≤ 0.5% |
| Kafka Consumer Lag | 정상범위 유지(스파이크 후 회복) |
| 재고 정합성 | 동시요청에도 초과판매 0건 |

## 2. 필수 시나리오
- 스파이크: 오픈 순간 대량 동시 진입(대기열 분산 검증)
- 매진 경합: 잔여 N석에 다수 동시 HOLD → 초과판매 0 확인
- 지속 부하: 일정 TPS 유지 시 p95/Lag 안정성
- 실패 주입: PG/Kafka 지연·실패 시 DLQ 적재·재처리

## 3. 합격 판정(PASS/FAIL)
- 위 기준 임계 충족 + 초과판매 0 + DLQ 정상 재처리 → PASS
- 결과는 개발자 화면(D-LOADTEST-RESULT)에 표시, 리포트로 전후 비교

## 4. 분리 원칙
- 성능 회귀는 하네스가 아니라 **부하테스트 기준**으로 환류(feedback-routing).
